Detailed work

As mentionned in the video, we have had to face a huge amount of data (approximately 8GB). We have been working with three different .json files:

  • one containing the business data including location data, attributes, and categories. (business.json)
  • one containing full review text data including the user_id that wrote the review and the business_id the review is written for. (review.json)
  • one containing the user data including the user's friend mapping and all the metadata associated with the user. (user.json)

    • We have chosen that dataset because it is more relevant for us to rank business according to comments than according to "stars". Actually, a user can grade a business by giving it an certain amount of stars without really how relevant his mark is going to be. When a user leave a comment he takes some time to explain his feeling in his own words. Then we analyze that comment to end up with a "happiness mark".

    That have lead us to use new tools in order to filter and to parse the collected data. To do so, we have used the Spark Python API (PySpark) which is a fast and general engine for large-scale data processing. Thanks to it we have been able to reduce the files' size regarding relevant filters.
